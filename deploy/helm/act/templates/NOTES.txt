A pre-install/pre-upgrade Job (services/api/src/bin/migrate.ts) applies schema
migrations before the API Deployment rolls out. If it fails, the release

  kubectl logs -n {{ .Release.Namespace }} job/{{ include "act.fullname" . }}-migrate-{{ .Release.Revision }}

Check readiness (verifies real connectivity to the configured storage backend):

  kubectl exec -n {{ .Release.Namespace }} deploy/{{ include "act.fullname" . }}-api -- \
    node -e "fetch('http://127.0.0.1:{{ .Values.api.port }}/v1/health/ready').then(r=>r.text()).then(console.log)"
